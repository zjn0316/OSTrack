# 问题及解决措施

## 问题

- 现有 `uwb_tcn_residual_seq10_ep10_prune_test.yaml` 是剪枝测试配置，不能清晰区分非 CE 与 CE 两组实验。
- Stage-2 需要同时加载 Stage-1 UWB checkpoint 与 OSTrack 视觉 checkpoint，两个来源不同。
- Stage-2 使用 ViT 视觉分支后，`BATCH_SIZE=256` 会在 8GB GPU 上显存溢出。

## 解决措施

- 新增两个独立 YAML，分别用于 Stage-2 非 CE 与 CE 训练。
- Stage-1 权重通过训练命令的 `--script_prv ugtrack --config_prv uwb_tcn_residual_seq10_ep100_l1_bce05` 加载。
- OSTrack 视觉权重通过 YAML 中 `MODEL.PRETRAIN_FILE` 加载。
- 将快速闭环配置改为 `BATCH_SIZE=4`，并降低 `SAMPLE_PER_EPOCH`，先确认训练、保存权重、测试、结果分析链路可用。
- 训练与测试结束时出现 `wandb` 临时目录清理权限警告，但主进程 exit code 为 0，checkpoint 与测试结果均已正常生成。
