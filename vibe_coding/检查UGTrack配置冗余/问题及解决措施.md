# 问题及解决措施

## 2026-05-01

- 问题：`lib/config/ugtrack/config.py` 中可能存在从 OSTrack 继承但 UGTrack 当前代码未使用的配置项。
- 措施：使用 `rg` 检查配置字段在 `lib/`、`tracking/`、`experiments/`、`doc/` 中的引用情况。
- 结论：`TRAIN.SCHEDULER.DECAY_RATE` 未被 UGTrack 代码读取；当前 UGTrack 仅支持 `TRAIN.SCHEDULER.TYPE="step"`，学习率衰减时机由 `TRAIN.LR_DROP_EPOCH` 控制。
- 处理：删除 `TRAIN.SCHEDULER.DECAY_RATE` 并在 `doc/UGTrack/YAML说明.md` 记录该冗余项。
