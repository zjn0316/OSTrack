# 问题及解决措施

| 问题 | 影响 | 解决措施 |
| --- | --- | --- |
| 需求要求读取 `uwb_obs.txt` 和 `uwb_conf.txt`，现有代码读取 `uwb_noise.txt` 和 `alpha_gt.txt` | 数据语义与论文需求不一致 | 设计文档中提出优先读取新文件、旧文件兼容回退的迁移方案 |
| 需求要求 UWB 编码器输出 128 维并支持 MLP/GRU/TCN，现有代码默认输出 768 维并支持 MLP/Conv1D | 模型复杂度和结构与需求不一致 | 设计文档中提出重构 UWB encoder 与多任务 UWBHead |
| Stage-1 当前使用 MSE 训练位置与 alpha，需求要求 L1 和 BCEWithLogitsLoss | 损失函数不符合 `uwb_conf` 软标签监督方案 | 设计文档中明确 Stage-1 损失调整方式 |
| Stage-2 当前冻结整个 UWB branch 的粒度过粗 | `uwb_token_head` 无法在视觉任务中适配 | 设计文档中提出冻结 prior、训练 token head 的拆分策略 |
| CE backbone 当前未接入 `uwb_token`，也没有第 0 层剪枝 | 无法完成剪枝版 UGTrack | 设计文档中提出先落地非 CE UWB token，再新增 `uwb_pruning.py`，最后兼容 CE |
