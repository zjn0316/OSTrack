# 第三章 基于时空UWB与视觉融合的目标跟踪方法

## 3.1 总体框架

本文提出一种融合超宽带（UWB）时序信号与视觉信息的目标跟踪方法，整体架构如图3.1所示。给定模板帧与搜索帧的RGB图像，以及长度为 $T$ 的UWB时序序列（每帧包含经坐标变换后的2D投影坐标），方法首先通过时空UWB编码器提取融合特征，并输出置信度 $\alpha$ 与校正后的投影点；随后在Vision Transformer（ViT）的入口层执行UWB引导的搜索区域剪枝，根据 $\alpha$ 动态保留部分搜索token；在ViT各层中，利用通道注意力机制以 $\alpha$ 为条件调制UWB token的特征表示；最后，预测头输出目标边界框。整个框架采用两阶段独立训练策略：第一阶段冻结视觉分支，仅训练UWB编码器；第二阶段固定UWB编码器参数，仅训练视觉分支（包括ViT和预测头）。UWB编码器在推理时提供剪枝中心与置信度，不参与跟踪损失的梯度反传，从而避免不可微的剪枝操作对联合训练的影响。

## 3.2 时空UWB编码与先验注入

### 3.2.1 时空UWB编码器

为融合UWB定位序列的时空信息，本文设计基于一维卷积的 **ConvEncoder** 作为UWB编码器。其以对齐后的UWB坐标序列 $\mathbf{U} \in \mathbb{R}^{T \times 2}$ 为输入，提取全局特征 $\mathbf{f}_{\text{uwb}} \in \mathbb{R}^{768}$，并输出UWB token、置信度 $\alpha$、校正坐标 $(u_{\text{pred}}, v_{\text{pred}})$。输入序列归一化至 $[0,1]$，不足 $T$ 时重复最后一帧填充。

#### ConvEncoder（时序卷积）

```
输入: U ∈ ℝ^{T×2}
   │
   ▼ Conv1D(2→32, k=3, s=1) + BN+ReLU → T×32
   │
   ▼ Conv1D(32→64, k=3, s=1) + BN+ReLU → T×64
   │
   ▼ Conv1D(64→128, k=3, s=1) + BN+ReLU → T×128
   │
   ▼ Global Avg Pooling → 128
   │
   ▼ Linear(128→768) → f_uwb (768)
```

#### 多分支输出

```
                    f_uwb (768)
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
   ┌──────────┐   ┌────────────┐   ┌────────────┐
   │UWB token │   │ 置信度分支  │    │  投影分支   │
   │ reshape  │   │Linear→64   │   │Linear→64   │
   │ 1×768    │   │  + ReLU    │   │  + ReLU    │
   └──────────┘   │Linear→1    │   │Linear→2    │
                  │ + Sigmoid  │   │ + Sigmoid  │
                  │     ↓      │   │     ↓      │
                  │  α ∈ [0,1]│   │(u_pred,    │
                  └────────────┘   │ v_pred)    │
                                   └────────────┘
```

### 3.2.2 UWB 引导的预处理：搜索区域剪枝与 UWB token 调制

传统 ViT 跟踪器对所有搜索 patch 进行自注意力计算，计算量与搜索区域大小呈正比。为降低冗余背景 token 的计算并有效利用 UWB 空间与质量信息，本文在 ViT **第 0 层**（patch embedding 与位置编码之后、第一个 Transformer block 之前）依次执行两项操作：**UWB 引导的搜索区域剪枝** 与 **基于通道注意力的 UWB token 调制**。二者共享同一 UWB 置信度 $\alpha$，先剪枝减少 token 数量，再调制增强 UWB token 的可靠性，共同为后续的 Transformer 层提供紧凑且可信的输入序列。

#### 1. UWB 引导的搜索区域剪枝

搜索图像经 patch embedding 后得到 $16\times16 = 256$ 个 token（每个 token 对应 $16\times16$ 像素的 patch）。每个 patch 的中心归一化坐标 $(c_x^{(i)},c_y^{(i)})$ 可预先计算并存储为常量。给定 UWB 预测的投影点 $(u_{\text{pred}},v_{\text{pred}})$（归一化）及其置信度 $\alpha \in [0,1]$，首先确定保留比例：
$$
r = 1 - 0.3\alpha,\quad r\in[0.7,1.0]
$$
保留的 token 数量为 $K = \lceil 256 \times r \rceil$（向上取整保证至少保留整数个 token）。  
接着，计算每个搜索 patch 的高斯空间得分：
$$
s_i = \exp\left(-\frac{(c_x^{(i)}-u_{\text{pred}})^2 + (c_y^{(i)}-v_{\text{pred}})^2}{2\sigma^2}\right),\quad \sigma = 0.125
$$
其中 $\sigma = 32/256$，对应图像空间中 32 像素的半径——该值经验覆盖了目标在相邻帧间的典型位移范围，且与 patch 尺寸（16 像素）协调。根据得分进行 Top‑K 选择，保留得分最高的 $K$ 个 token，其余丢弃。剪枝后的搜索 token $\mathbf{X}_{\text{search}}^{\text{pruned}} \in \mathbb{R}^{K \times D}$ 与模板 token（64 个）、新引入的 UWB token（1 个）拼接，形成完整的 ViT 输入序列。本文固定采用如下拼接顺序：
$$
[\text{Template Tokens}]\,+\,[\text{UWB Token}]\,+\,[\text{Search Tokens}]
$$
其中模板 token 与 UWB token 构成前缀组，搜索 token 作为后缀组连续排列。

**位置编码策略**：采用可学习的绝对位置编码。具体实现如下：

 - **模板 token**：共64个，每个token对应一个可学习的 $D$ 维向量，参数维度为 $64 \times D$，与OSTrack相同。
 - **搜索 token**：原始搜索图像共 $16\times16=256$ 个patch，预先为每个patch索引 $i\in\{1,\dots,256\}$ 分配一个可学习的 $D$ 维位置编码，组成矩阵 $\mathbf{P}_{\text{search}} \in \mathbb{R}^{256\times D}$。剪枝时，模型根据高斯得分计算保留的patch索引集合 $\mathcal{I}_{\text{keep}}$（大小为 $K$）保留对应的搜索token特征，并**根据其原始索引**从 $\mathbf{P}_{\text{search}}$ 中取出对应的位置编码与之相加。由于剪枝操作仅删除部分token而不改变剩余token的相对顺序，这种**按原始索引直接取值**的方式保证了每个token的位置编码与它在图像中的实际空间坐标严格对齐。
 - **UWB token**：仅1个，使用单独的可学习位置编码 $\mathbf{p}_{\text{uwb}} \in \mathbb{R}^{D}$。

所有位置编码参数在阶段二（视觉分支微调）中与ViT参数共同优化。需要说明的是，剪枝操作本身不参与梯度反传（即被删除token对应的位置编码不更新），但保留token的位置编码仍可通过跟踪损失的梯度正常更新。该设计完全沿用了OSTrack官方代码中的位置编码与token剪枝的交互逻辑

#### 2. 基于通道注意力的 UWB token 调制

UWB 信号质量易受环境遮挡、多径效应等影响。为避免低质量信号干扰视觉特征学习，在剪枝之后、进入第一个 Transformer block 之前，本文引入一个轻量级通道注意力模块，以同一置信度 $\alpha$ 为条件动态调制 UWB token 的特征。之所以仅在 **第 0 层** 调制（而非每层），原因如下：

- **置信度的全局静态性**：UWB 输出的 $\alpha$ 在单次跟踪中变化极慢，深层调制不会带来额外收益；
- **浅层融合最敏感**：第 0 层是模板、搜索、UWB token 首次交互的位置，在此处显式调制 UWB token 效果最直接；
- **避免过拟合**：每层引入注意力会大幅增加参数量（$\approx 2D\times64$ 每层），而单层调制足以实现软性自适应。

设当前层 UWB token 的特征为 $\mathbf{t}_{\text{uwb}} \in \mathbb{R}^{1 \times D}$。首先将标量 $\alpha$ 通过两层 MLP 映射为通道权重向量 $\mathbf{w} \in \mathbb{R}^D$：

$$
\mathbf{w} = \text{Sigmoid}\big(\mathbf{W}_2 \cdot \text{ReLU}(\mathbf{W}_1 \cdot \alpha + \mathbf{b}_1) + \mathbf{b}_2\big)
$$

其中 $\mathbf{W}_1 \in \mathbb{R}^{64 \times 1}$，$\mathbf{W}_2 \in \mathbb{R}^{D \times 64}$。然后对 UWB token 逐通道加权：

$$
\tilde{\mathbf{t}}_{\text{uwb}} = \mathbf{t}_{\text{uwb}} \odot \mathbf{w}
$$

调制后的 $\tilde{\mathbf{t}}_{\text{uwb}}$ 替代原 token 参与后续自注意力计算。

**学习机制**：该调制模块的所有参数（$\mathbf{W}_1,\mathbf{b}_1,\mathbf{W}_2,\mathbf{b}_2$）与视觉分支联合训练，仅依靠跟踪损失（GIoU + L1 + Focal）的梯度进行更新，无需额外监督信号。网络通过最小化损失自动探索：当 $\alpha$ 较大（信号可信）时，抑制 UWB token 会损害跟踪精度，梯度推动 $\mathbf{w}$ 接近全 1；当 $\alpha$ 较小时，UWB 测量可能引入噪声，网络学会将 $\mathbf{w}$ 推向 0 以忽略该 token。因此，$\alpha$ 与 $\mathbf{w}$ 之间的正相关性是数据驱动的隐式学习结果，而非人为硬编码。

#### 3. 与 CE 剪枝的协同

本文沿用原始 OSTrack 中的 CE 剪枝策略（基于注意力得分保留得分最高的前 70% token），在 ViT 的第 4、7、10 层执行（实现配置记作 $\text{CE\_LOC}=[3,6,9]$，与 0-index 记法一一对应）。由于 UWB 引导的剪枝与调制均在第 0 层完成，二者与深层 CE 机制互不干扰：UWB 预处理先产生紧凑的 token 序列，且序列组织始终满足“前缀组（模板+UWB）+后缀组（搜索）”，后续 CE 层仅操作搜索 token（UWB token 始终保留），无需再考虑 UWB 的空间先验。该协同设计在保证效果的同时最大化计算效率。

## 3.3 不确定性感知的自适应融合

### 3.3.1 置信度监督与投影校正

为利用时序信息提升对当前帧UWB信号质量的估计鲁棒性，本文基于真实UWB数据离线统计得到的噪声分布，在训练阶段构造单帧置信度硬标签 $\alpha_{\text{gt}}$。具体构造方法如下：

1. **理想投影点**：将公开数据集中目标真实中心 $(u_{\text{gt}}, v_{\text{gt}})$（归一化坐标）视为理想UWB投影点。

2. **生成带噪序列**：对该理想点，根据真实UWB数据统计得到的噪声分布（如经验分布或非参数统计特征）独立采样 $T$ 次，得到长度为 $T$ 的带噪投影序列 $\{(u_{\text{noise}}^{(t)}, v_{\text{noise}}^{(t)})\}_{t=1}^{T}$。其中前 $T-1$ 帧作为历史上下文，第 $T$ 帧作为当前时刻的UWB观测。

3. **当前帧置信度标签**：由当前帧投影点与真值的欧氏距离计算瞬时置信度：
   $$
   \alpha_{\text{gt}} = \exp\left(-\frac{\|(u_{\text{noise}}^{(T)}, v_{\text{noise}}^{(T)}) - (u_{\text{gt}}, v_{\text{gt}})\|_2^2}{2\sigma_0^2}\right)
   $$
   其中 $\sigma_0 = 10/256$，使得当投影误差为10像素时，$\alpha_{\text{gt}} \approx 0.6065$。该标签直接反映当前帧UWB测量的可靠程度。

模型以T帧UWB投影点为输入，预测当前帧的置信度 $\alpha$ 以及当前帧的校正后投影点 $(u_{\text{pred}}, v_{\text{pred}})$。训练损失包括：

- 置信度预测的均方误差损失：
  $$
  \mathcal{L}_{\alpha}=\mathrm{MSE}(\alpha,\alpha_{\text{gt}})=(\alpha-\alpha_{\text{gt}})^2
  $$

- 投影校正点的均方误差损失：
  $$
  \mathcal{L}_{\text{proj}} = \text{MSE}\big((u_{\text{pred}}, v_{\text{pred}}), (u_{\text{gt}}, v_{\text{gt}})\big)
  $$

### 3.3.2 两阶段训练策略

为避免不可微的剪枝操作对训练造成影响，本文采用两阶段独立训练策略：  

 **阶段一：UWB编码器预训练**。固定视觉分支参数（使用预训练的OSTrack权重，不更新），仅训练UWB编码器。损失函数为：  
$$
 \mathcal{L}_{\text{pre}} = \lambda_{\text{proj}} \mathcal{L}_{\text{proj}} + \lambda_{\alpha} \mathcal{L}_{\alpha}
$$
 其中 $\mathcal{L}_{\text{proj}}$ 为投影校正的MSE损失，$\mathcal{L}_{\alpha}$ 为置信度的MSE损失。此阶段UWB编码器快速收敛至较准确的投影校正与置信度预测。  

 **阶段二：视觉分支微调**。加载预训练好的UWB编码器权重并**固定**（不参与梯度更新），仅训练视觉分支（包括ViT、预测头以及通道注意力调制模块中的 $\mathbf{W}_1,\mathbf{W}_2$ 等参数）。此时通道注意力使用阶段一预测的 α，由于α已提供合理的先验，视觉分支可稳定地学习如何利用UWB token进行剪枝与调制。阶段二仅使用跟踪损失 $\mathcal{L}_{\text{track}}$ 进行训练。由于 UWB 编码器已固定，其输出的投影点和置信度不再变化，因此无需再计算 $\mathcal{L}_{\text{proj}}$ 和 $\mathcal{L}_{\alpha}$。

通过这种设计，UWB编码器的输出仅作为推理时的先验，无需进行端到端梯度反传，从而完全规避了硬剪枝不可微的问题。

### 3.3.3 推理时的自适应融合

推理阶段无需任何显式阈值或切换逻辑，全流程连续自适应：

- 根据预测的 $\alpha$ 计算保留比例 $r = 1 - 0.3\alpha$，执行第0层剪枝。
- 在第0层（第一个Transformer block之前），通道注意力模块根据$\alpha$ 调制UWB token。

这种设计使得模型能够根据UWB信号的实时质量平滑地调整融合强度，无需人工设定降级阈值。

## 3.4 损失函数汇总

结合两阶段训练策略，本文采用**分阶段损失定义**：

- **阶段一（UWB编码器预训练）**：
  $$
  \mathcal{L}_{\text{stage1}}=\lambda_{\text{proj}}\mathcal{L}_{\text{proj}}+\lambda_{\alpha}\mathcal{L}_{\alpha}
  $$

- **阶段二（视觉分支微调，UWB编码器冻结）**：
  $$
  \mathcal{L}_{\text{stage2}}=\mathcal{L}_{\text{track}}=\lambda_{\text{giou}} \mathcal{L}_{\text{GIoU}} + \lambda_{\text{l1}} \mathcal{L}_{\text{L1}} + \lambda_{\text{focal}} \mathcal{L}_{\text{focal}}
  $$

各损失项的作用如下：

- $\mathcal{L}_{\text{track}}$：标准跟踪损失，监督边界框回归与分类。
- $\mathcal{L}_{\text{proj}}$：投影校正损失，引导UWB编码器输出准确的目标中心投影。
- $\mathcal{L}_{\alpha}$：置信度损失，使网络学会评估UWB信号质量。

通过这种分阶段训练方式，阶段一先学习稳定的UWB投影与置信度估计，阶段二在固定UWB先验的条件下优化视觉跟踪主任务，从而规避硬剪枝不可微带来的训练不稳定问题。
