# Stage-1 UWB 分支实验结果

## 实验设置

- **全因子设计**：3 编码器（MLP / GRU / TCN）× 4 seq_len（1/3/5/10）× 4 conf weight（0.1/0.25/0.5/1.0）= **48 组实验**
- **评估数据集**：OTB100_UWB（20 序列，10238 帧，遮挡 6.69%）、UAV123_UWB、custom_dataset
- **搜索区域**：256×256
- **固定参数**：UWB_PRED_MODE=residual, BATCH_SIZE=256, NUM_WORKER=8, EPOCH=100, SAMPLE_PER_EPOCH=5000, VAL_SAMPLE_PER_EPOCH=1000, LR=0.001, WEIGHT_DECAY=0.0001
- **评估维度**：All（全体）/ Non-occ（非遮挡）/ Occ（遮挡）

#### UAV123_UWB：强泛化，但部分配置在遮挡场景崩溃

UAV123_UWB 的非遮挡 uv_MAE_px 全部在 0.4~0.9px（OTB100_UWB 为 1.6px），说明模型在该数据集上的非遮挡预测非常精准。但遮挡场景呈现高度不稳定性：

**稳定配置（Occ uv_MAE_px < 5px）：**

| 配置 | Occ uv_MAE_px | 类型 |
|---|---|---|
| MLP_T5_W01 | **0.75** | 非遮挡也稳（0.65px） |
| MLP_T5_W025 | **0.74** | 非遮挡也稳（0.62px） |
| GRU_T3_W05 | **0.85** | 全局最优 All 0.49px |
| GRU_T5_W025 | **0.62** | GRU 遮挡最佳 |
| GRU_T5_W01 | **1.05** | 稳定 |
| TCN_T1_W01 | **0.76** | TCN 最佳表现 |
| GRU_T1_W05 | 0.50 | 非遮挡 0.56px，但 Occ 实际稳定 |

**不稳定配置（Occ uv_MAE_px > 80px，完全崩溃）：**

| 配置 | Occ uv_MAE_px | 崩溃模式 |
|---|---|---|
| MLP_T1_W025 | **92.67** | MLP T1 高权重崩溃 |
| MLP_T1_W05 | **122.58** | 最严重 |
| MLP_T3_W05 | **82.98** | MLP 在 OTB100 上最优，在 UAV123 Occ 上崩溃 |
| MLP_T5_W05 | **101.50** | T5+W0.5 组合在 Occ 上不稳定 |

**UAV123 遮挡崩溃的规律：**
1. **MLP + W=0.5 最危险**：MLP_T3_W05（OTB100 最优配置）在 UAV123 Occ 上 uv_MAE_px=82.98，完全不可用
2. **GRU 整体最稳健**：GRU T3/T5 的 Occ uv_MAE_px 全部 < 2.3px（除 T5_W05 的 9.89px 和 T3_W10 的 15.49px）
3. **低权重（W=0.1/0.25）更可靠**：几乎所有崩溃案例均发生在 W=0.5 或 W=1.0
4. **TCN T1 意外稳健**：TCN_T1_W01/W025/W05 的 Occ 仅为 0.76~0.81px，是所有编码器中 Occ 最稳定的

**GRU 的稳健性分析：**
GRU 在 UAV123 上展现出独特的跨数据集稳健性。以 GRU_T3_W05 为例：
- OTB100 All: 2.176px（中等水平）
- UAV123 All: **0.486px**（全场最佳）
- UAV123 Occ: 0.849px（第二佳）
- custom_dataset All: 46.03px（所有配置均差，非 GRU 特有）

这表明 GRU 的循环结构在数据分布偏移时仍能保持稳定的隐状态表征，不像 MLP 那样出现极端预测。

#### custom_dataset：跨数据集泛化失效（模拟→真实 UWB）

所有 48 个配置在 custom_dataset 上的 uv_MAE_px 为 41~46px（All），远超 OTB100（2.1px）和 UAV123（0.5px）。这并非坐标系统差异导致。

**两个数据集的 UWB 坐标都在原始图像空间中：**

| 项目 | OTB100_UWB | custom_dataset |
|---|---|---|
| UWB 坐标范围 | U 11.6~594.3, V 63.5~358.3 | U 0.0~453.0, V 30.0~191.5 |
| 图像尺寸 | 640×480 | 640×360 |
| 坐标语义 | 原始图像像素坐标 | 原始图像像素坐标 |

两者坐标范围可比（OTB100 U 方向范围 594 甚至大于 custom_dataset 的 453），训练时统一通过 `transform_uwb_to_crop()` 转换到搜索裁剪归一化空间，路径完全一致。

**根因：模拟 UWB vs 真实 UWB 的域偏移**

预处理流程对比：

| 处理步骤 | OTB100_UWB | custom_dataset |
|---|---|---|
| UWB 观测来源 | **模拟生成**（高斯噪声+漂移+离群点） | **真实硬件采集** |
| uwb_obs 生成 | uwb_gt + noise + drift + outlier | 原始 uwb.txt 取前两列 (u,v) |
| uwb_gt 生成 | bbox 中心 → 遮挡感知双向 EMA 平滑 | bbox 中心 → 缺失段线性插值 |
| uwb_conf 生成 | 基于 obs-gt 距离 + bbox 尺度 + 多种因素 | 基于 obs-gt 距离 + valid/occlusion 调制 |

两个数据的预处理链路结构相似（bbox→uwb_gt→uwb_obs→uwb_conf），但**最关键的差异在于 uwb_obs 的来源**：

- **OTB100 的 uwb_obs 是模拟的**：噪声模型为高斯分布 + 可控漂移 + 稀疏离群点，噪声参数与 bbox 短边成比例（`sigma = 0.06~0.14 * bbox_short`），有明确的数学形式
- **custom_dataset 的 uwb_obs 是真实的**：包含真实环境中的多径效应、信号衰减、硬件非理想性等复杂因素，噪声分布远非高斯模型能描述

模型**仅在 OTB100 的模拟 UWB 上训练**，当遇到 custom_dataset 的真实 UWB 数据时，观测噪声模式完全不同，导致定位预测失控。

**解释 ConfPear 高（0.91~0.97）：**
置信度预测学习的是"UWB 观测噪声水平 → 置信度"的映射关系。custom_dataset 的 uwb_conf 基于 obs-gt 距离生成，噪声大的帧置信度低——这种信噪比关系在不同 UWB 设备间具有通用性。模型能正确判断"信号质量好的帧 → 高置信度"，即使具体的位置预测因域偏移而偏离。因此 ConfPear 高（置信度排序正确）与 uv_MAE 大（定位偏差大）并不矛盾。

综合三个数据集的表现，对候选配置进行稳定性评估：

### 5. 结论与推荐

#### 默认推荐（以 OTB100 为主数据集）

| 参数 | 选择 | 理由 |
|---|---|---|
| **编码器** | **MLP** | OTB100 定位最优（2.105px），参数量最小 |
| **seq_len** | **3** | MLP 在 T=3 达峰值，T=1 无置信度，T≥5 开始退化 |
| **conf weight** | **0.5** | MLP T3 下所有权重均稳定，选中间值确保泛化 |

#### 跨数据集推荐（需处理遮挡 + 未知数据集）

| 参数 | 选择 | 理由 |
|---|---|---|
| **编码器** | **GRU** | 跨数据集最稳健，UAV123 Occ 不崩溃 |
| **seq_len** | **3** | GRU T3 在 UAV123 上 All 0.486px / Occ 0.849px |
| **conf weight** | **0.5** | GRU T3 下 W=0.5 在 UAV123 Occ 仅 0.85px |
| **最终配置** | **GRU_T3_W05** | — |

#### 替代方案

| 场景 | 推荐配置 | 理由 |
|---|---|---|
| 仅 OTB100 部署 | MLP_T3_W05 | 2.105px 最优定位 |
| 跨数据集 + 遮挡频繁 | GRU_T3_W05 | UAV123 Occ 0.85px，无崩溃风险 |
| 跨数据集 + 遮挡极少 | MLP_T5_W01 | UAV123 All 0.65px / Occ 0.75px，MLP 的轻量优势 |
| 置信度质量优先 | MLP_T3_W10 | ConfPear=0.278，定位 2.173px |
