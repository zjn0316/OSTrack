建议按“先验证 UWB prior，再验证 token 融合，再验证 0 层剪枝”的顺序做，不要一开始铺太大。

**第一组：Stage-1 UWB 分支实验**
目的：确认 UWB encoder 的位置预测和置信度预测可靠。

| 实验 | Encoder | SEQ_LEN | pred_mode | loss | 目的 |
|---|---:|---:|---|---|---|
| S1-MLP-10 | mlp | 10 | residual | L1 + BCE | 轻量基线 |
| S1-GRU-10 | gru | 10 | residual | L1 + BCE | 时序建模对比 |
| S1-TCN-10 | tcn | 10 | residual | L1 + BCE | 主实验候选 |
| S1-TCN-5 | tcn | 5 | residual | L1 + BCE | 短历史对比 |
| S1-TCN-20 | tcn | 20 | residual | L1 + BCE | 长历史对比 |

优先级：先做 `TCN seq10`，再做 `MLP/GRU seq10`，最后做 `seq_len` 消融。

指标：

```text
UWB position error
uv_pred_auc
conf_auc
occlusion_auc
Loss/uwb_pred
Loss/uwb_conf
```

**第二组：Stage-2 主跟踪实验**
目的：确认 UWB token 和 0 层剪枝是否提升跟踪。

| 实验 | Backbone | UWB token | 0层剪枝 | CE | 目的 |
|---|---|---:|---:|---:|---|
| OSTrack baseline | 原 OSTrack | 否 | 否 | 否 | 纯视觉基线 |
| UGTrack-token | vit_uwb | 是 | 否 | 否 | 验证 UWB token 融合 |
| UGTrack-token-prune | vit_uwb | 是 | 是 | 否 | 验证 0 层剪枝 |
| UGTrack-token-CE | vit_ce_uwb | 是 | 否 | 是 | 验证 CE 兼容 |
| UGTrack-token-prune-CE | vit_ce_uwb | 是 | 是 | 是 | 完整模型 |

优先级最高的是这三个：

```text
OSTrack baseline
UGTrack-token
UGTrack-token-prune
```

如果这三个没跑通，先不要扩 CE。

**第三组：0 层剪枝消融**
目的：确认剪枝强度是否合理。

建议固定 Stage-1 最优 checkpoint，例如 `TCN seq10`，只改剪枝参数：

| 实验 | dynamic | min_keep | max_keep | fixed_keep | 说明 |
|---|---:|---:|---:|---:|---|
| no-prune | - | - | - | - | 不启用剪枝 |
| fixed-0.75 | False | - | - | 0.75 | 保守剪枝 |
| fixed-0.50 | False | - | - | 0.50 | 中等剪枝 |
| fixed-0.25 | False | - | - | 0.25 | 激进剪枝 |
| dynamic-0.25-1.0 | True | 0.25 | 1.0 | - | 当前默认 |
| dynamic-0.50-1.0 | True | 0.50 | 1.0 | - | 更保守动态剪枝 |

重点看：

```text
Success AUC
Precision
Normalized Precision
FPS
uwb_prune_keep_tokens
uwb_prune_keep_ratio
```

**第四组：关键消融**
这些用于论文表格：

| 消融 | 设置 | 目的 |
|---|---|---|
| no UWB | 原 OSTrack | 视觉基线 |
| UWB token only | 只加 token，不剪枝 | 验证跨模态融合 |
| prune only | 用 UWB pred/conf 剪枝，但不加 UWB token | 验证空间先验 |
| token + prune | 完整 UGTrack | 验证组合效果 |
| freeze prior vs trainable prior | Stage-2 冻结/不冻结 UWB prior | 验证两阶段策略必要性 |
| residual vs direct | UWB_PRED_MODE | 验证坐标预测方式 |

其中 `prune only` 可能需要小改代码，让 `uwb_token=None` 但仍传 `pred_uv/uwb_conf_pred` 做剪枝；不是第一优先级。

**推荐最小实验路线**
先跑这 6 个就够判断方向：

```text
1. Stage-1: TCN seq10 residual
2. Stage-1: MLP seq10 residual
3. Stage-1: GRU seq10 residual
4. Stage-2: OSTrack baseline
5. Stage-2: UGTrack-token
6. Stage-2: UGTrack-token-prune
```

如果 `UGTrack-token` 比 baseline 有提升，再扩 `CE` 和剪枝强度消融；如果 token 没提升，优先查 Stage-1 prior 质量和 UWB token head 训练策略。
