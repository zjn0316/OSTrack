# 加载指定权重

本文说明在当前工程中，如何加载某个脚本、某个配置、某一轮已经存在的 checkpoint。

## 1. checkpoint 命名规则

训练权重默认保存到：

```text
<save_dir>/checkpoints/train/<script>/<config>/<NetClass>_epXXXX.pth.tar
```

常见示例：

```text
output/checkpoints/train/ostrack/vitb_256_mae_32x4_ep300/OSTrack_ep0300.pth.tar
output/checkpoints/train/ugtrack/uwb_conv1d/UGTrack_ep0300.pth.tar
output/checkpoints/train/ugtrack/vitb_256_mae_ep300_uwb_conv1d_token/UGTrack_ep0300.pth.tar
```

其中：

- `<script>` 来自训练命令的 `--script`
- `<config>` 来自训练命令的 `--config`
- `XXXX` 是 epoch，四位补零，例如第 40 轮是 `ep0040`

## 2. 权重加载代码位置

权重加载通常分两步：

```text
1. 根据配置、命令行参数或 checkpoint 目录确定权重文件路径。
2. 用 torch.load() 读取 checkpoint，再用 load_state_dict() 写入模型。
```

### 2.1 MAE ViT 权重加载到 backbone

路径确定位置：

```text
lib/models/ugtrack/ugtrack.py
```

`build_ugtrack()` 读取 `cfg.MODEL.PRETRAIN_FILE`。如果它不是 OSTrack tracker checkpoint，就拼到：

```text
pretrained_models/<PRETRAIN_FILE>
```

然后传给 ViT 构建函数：

```text
vit_base_patch16_224(mae_pretrained, ...)
vit_base_patch16_224_ce(mae_pretrained, ...)
vit_large_patch16_224_ce(mae_pretrained, ...)
```

真正加载位置：

```text
lib/models/ostrack/vit.py
```

核心代码：

```python
checkpoint = torch.load(pretrained, map_location="cpu")
missing_keys, unexpected_keys = model.load_state_dict(checkpoint["model"], strict=False)
```

加载内容：

```text
MAE 预训练 ViT backbone 权重。
不加载 tracking head，因为 MAE checkpoint 没有 OSTrack head。
```

### 2.2 OSTrack tracker 权重加载到 UGTrack tracker

路径确定位置：

```text
experiments/ugtrack/vitb_256_mae_ep300_ostrack_uwb_conv1d_token.yaml
```

配置项：

```yaml
MODEL:
  PRETRAIN_FILE: output/checkpoints/train/ostrack/vitb_256_mae_32x4_ep300/OSTrack_ep0300.pth.tar
```

真正加载位置：

```text
lib/models/ugtrack/ugtrack.py
```

核心代码：

```python
checkpoint = torch.load(pretrain_file, map_location="cpu")
missing_keys, unexpected_keys = tracker.load_state_dict(checkpoint["net"], strict=False)
```

加载内容：

```text
已训练 OSTrack tracker 权重，包括 ViT/backbone 和 box head。
```

### 2.3 Stage-1 UWB checkpoint 加载到 Stage-2 UGTrack

路径确定位置：

```text
tracking/train.py
lib/train/run_training.py
```

命令行参数：

```powershell
--script_prv ugtrack --config_prv uwb_conv1d
```

`run_training.py` 会设置：

```python
settings.project_path_prv = "train/{}/{}".format(script_name_prv, config_name_prv)
```

调用加载位置：

```text
lib/train/trainers/base_trainer.py
```

核心代码：

```python
directory = "{}/{}".format(self._checkpoint_dir, self.settings.project_path_prv)
self.load_state_dict(directory)
```

真正加载位置同文件：

```python
checkpoint_dict = torch.load(checkpoint_path, map_location="cpu")
missing_k, unexpected_k = net.load_state_dict(checkpoint_dict["net"], strict=False)
```

加载内容：

```text
Stage-1 checkpoint 中的 uwb_branch.* 权重。
Stage-1 checkpoint 没有 tracker，因此 Stage-2 加载时 tracker.* 出现在 missing keys 是正常现象。
```

### 2.4 当前实验 checkpoint 断点续训

路径确定位置：

```text
lib/train/trainers/base_trainer.py
```

核心代码：

```python
checkpoint_list = sorted(glob.glob("{}/{}/{}_ep*.pth.tar".format(
    self._checkpoint_dir, self.settings.project_path, net_type)))
checkpoint_path = checkpoint_list[-1]
```

真正加载位置：

```python
net.load_state_dict(checkpoint_dict[key])
self.optimizer.load_state_dict(checkpoint_dict[key])
```

加载内容：

```text
当前实验完整 checkpoint，包括 uwb_branch、tracker、optimizer、epoch、stats。
这是断点续训，不是初始化。
```

### 2.5 测试时加载 UGTrack checkpoint

路径确定位置：

```text
lib/test/parameter/ugtrack.py
```

真正加载位置：

```text
lib/test/tracker/ugtrack.py
```

核心代码：

```python
checkpoint = torch.load(self.params.checkpoint, map_location="cpu")
missing_keys, unexpected_keys = network.load_state_dict(checkpoint["net"], strict=False)
```

加载内容：

```text
测试时加载完整 UGTrack checkpoint，用于推理。
```

## 3. 测试时加载指定 epoch

测试侧最直接。修改对应 yaml 的：

```yaml
TEST:
  EPOCH: 300
```

例如想测试 UGTrack 第 240 轮：

```yaml
TEST:
  EPOCH: 240
```

然后运行：

```powershell
python tracking/test.py ugtrack vitb_256_mae_ep300_uwb_conv1d_token --dataset_name otb100_uwb --threads 0 --num_gpus 1
```

测试参数文件会自动定位：

```text
output/checkpoints/train/ugtrack/vitb_256_mae_ep300_uwb_conv1d_token/UGTrack_ep0240.pth.tar
```

注意：如果结果目录里已有同名序列结果，`tracking/test.py` 在非 debug 模式会跳过已存在结果。需要重新测时，先换 `--runid`，或手动处理对应 tracking result 文件。

## 4. 训练时从当前实验继续训练

当前训练脚本内部使用：

```python
trainer.train(..., load_latest=True, ...)
```

所以只要当前实验目录里已有 checkpoint，再次运行同一个：

```text
--script <script> --config <config>
```

就会自动加载当前实验目录下最新的 checkpoint。

例如：

```powershell
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_uwb_conv1d_token --save_dir ./output --mode single --use_wandb 0
```

会优先查找：

```text
output/checkpoints/train/ugtrack/vitb_256_mae_ep300_uwb_conv1d_token/UGTrack_ep*.pth.tar
```

并加载排序后最新的一个。

这个机制适合“断点续训”，不适合“从任意旧 epoch 重新开始一个新实验”。

当前约定：

```text
如果当前实验目录存在 checkpoint：
  优先断点续训当前实验；
  不再加载 --script_prv/--config_prv 指向的阶段一初始化权重。

如果当前实验目录不存在 checkpoint：
  按 MODEL.PRETRAIN_FILE 初始化视觉分支；
  Stage-2 再加载 --script_prv/--config_prv 指向的阶段一 UWB 权重。
```

因此，如果目标是“重新开始一个新实验”，不要复用已有 checkpoint 的 config 输出目录。

## 5. 训练时加载上一个阶段的最新权重

当前 UGTrack 阶段二训练使用：

```powershell
--script_prv ugtrack --config_prv uwb_conv1d
```

代码会定位：

```text
output/checkpoints/train/ugtrack/uwb_conv1d/
```

然后加载该目录下排序后最新的：

```text
UGTrack_ep*.pth.tar
```

示例：

```powershell
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_ostrack_uwb_conv1d_token --script_prv ugtrack --config_prv uwb_conv1d --save_dir ./output --mode single --use_wandb 0
```

这会把 `ugtrack/uwb_conv1d` 的最新 checkpoint 当作前一阶段权重加载。

## 6. 不改代码时，如何让 `--script_prv/--config_prv` 加载指定 epoch

当前命令行没有 `--epoch_prv` 参数，`--script_prv/--config_prv` 只能加载“目录里的最新 checkpoint”。

如果不想改代码，可以用“checkpoint 别名目录”方法：

1. 新建一个别名目录：

```text
output/checkpoints/train/ugtrack/uwb_conv1d_ep0100_alias/
```

2. 把想加载的 checkpoint 放进去，目录里只保留这一个 `*.pth.tar`：

```text
output/checkpoints/train/ugtrack/uwb_conv1d_ep0100_alias/UGTrack_ep0100.pth.tar
```

3. 训练时传这个别名配置名：

```powershell
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_ostrack_uwb_conv1d_token --script_prv ugtrack --config_prv uwb_conv1d_ep0100_alias --save_dir ./output --mode single --use_wandb 0
```

这里的 `config_prv` 只用于拼 checkpoint 路径，不会读取：

```text
experiments/ugtrack/uwb_conv1d_ep0100_alias.yaml
```

因此别名目录可以只用于指定权重来源。

## 7. 从 OSTrack 已训练权重初始化 UGTrack 视觉分支

如果目标是“在 OSTrack 基础上加入 UWB”，应该加载 OSTrack 的训练权重，而不是只用 MAE 预训练权重。

配置写法：

```yaml
MODEL:
  PRETRAIN_FILE: output/checkpoints/train/ostrack/vitb_256_mae_32x4_ep300/OSTrack_ep0300.pth.tar
```

注意：

```text
experiments/ostrack/vitb_256_mae_32x4_ep300.yaml
```

是配置文件，不是权重文件。真正的权重文件是：

```text
output/checkpoints/train/ostrack/vitb_256_mae_32x4_ep300/OSTrack_ep0300.pth.tar
```

推荐新建一个 UGTrack stage2 配置，例如：

```text
experiments/ugtrack/vitb_256_mae_ep300_ostrack_uwb_conv1d_token.yaml
```

然后在里面设置 `MODEL.PRETRAIN_FILE` 为 OSTrack checkpoint 路径。

## 8. 重要注意事项

如果当前实验目录已经存在 checkpoint，训练会先自动加载当前实验最新 checkpoint。为了确保“从指定权重重新训练”，建议使用新的 `--config` 名称，或确保当前实验输出目录为空。

例如不要复用旧的：

```text
vitb_256_mae_ep300_uwb_conv1d_token
```

而是使用新的：

```text
vitb_256_mae_ep300_ostrack_uwb_conv1d_token
```

这样可以避免旧实验 checkpoint 自动恢复，影响新实验起点。

训练前建议检查当前实验目录：

```powershell
Get-ChildItem .\output\checkpoints\train\ugtrack\<config_name>\*.pth.tar
```

如果这里已经有文件，再次运行同一 config 会被视为断点续训。

## 9. 推荐流程

### 场景 A：测试某个 epoch

1. 修改 yaml 中的 `TEST.EPOCH`
2. 运行 `tracking/test.py`
3. 运行 `tracking/analysis_results.py`

### 场景 B：阶段二加载阶段一最新权重

Stage-2 使用 MAE ViT 初始化视觉 backbone，head 随机初始化：

```powershell
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_uwb_conv1d_token --script_prv ugtrack --config_prv uwb_conv1d --save_dir ./output --mode single --use_wandb 0
```

### 场景 C：阶段二加载阶段一指定 epoch

1. 建立 checkpoint 别名目录
2. 目录中只放指定 epoch 的 checkpoint
3. `--config_prv` 指向这个别名目录名

### 场景 D：UGTrack 从 OSTrack 权重开始训练

1. 在 UGTrack stage2 yaml 中设置：

```yaml
MODEL:
  PRETRAIN_FILE: output/checkpoints/train/ostrack/vitb_256_mae_32x4_ep300/OSTrack_ep0300.pth.tar
```

2. 阶段一 UWB 权重仍通过 `--script_prv/--config_prv` 加载：

```powershell
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_ostrack_uwb_conv1d_token --script_prv ugtrack --config_prv uwb_conv1d --save_dir ./output --mode single --use_wandb 0
```

运行时应在日志中看到：

```text
UGTrack weight loading plan:
  stage: 2
  load_latest/current experiment resume: True
  load_previous/stage-1 init: True
  previous checkpoint project: train/ugtrack/uwb_conv1d
  model pretrain file: ...
```
