## 已完成步骤

- [x] 1. 已明确当前重构目标：在功能已实现的基础上，按模块逐个对齐 UGTrack 的代码组织和局部风格。
- [x] 2. 已完成 `lib/train/train_script_ugtrack.py` 对齐：训练入口保留配置加载、日志路径、dataloader、模型、DDP、Actor、optimizer、trainer 和 train 调用等装配流程，UGTrack 细节继续下沉到模型、Actor 和 base functions。
- [x] 3. 已检查并确认 `lib/config/ugtrack/config.py` 已对齐：配置结构沿用 OSTrack 的 `DATA / MODEL / TRAIN / TEST` 分区，UGTrack 的 UWB、stage、loss 权重等增量字段放入对应配置层级，`gen_config()` 与 `update_config_from_file()` 保持同类配置模块风格。
- [x] 4. 已完成 `lib/train/base_functions_ugtrack.py` 对齐：训练基础函数按职责拆分为 settings 更新、stage1 / stage2 dataloader 构建、数据集名称解析、UWB-only dataset、optimizer / scheduler 构建等模块，整体组织方式与 OSTrack `base_functions.py` 保持一致。
- [x] 5. 已完成 `lib/train/data` 数据流对齐：
  - `UWBProcessing` 按 `processing.py` 的主流程重写，主干为 `joint transform -> jitter bbox -> jittered_center_crop -> branch transform -> attention mask 检查 -> stack/squeeze`。
  - UWB 不再走“单独分支归一化”，而是与 bbox 共用像素坐标系，在 joint flip、crop 映射、search transform 中同步变换，最终输出为 search crop 内归一化坐标。
  - `UWBTrackingSampler` 保持与 `TrackingSampler` 相同的采样框架，只在 `TensorDict` 中额外放入 `search_uwb_seq`、`search_uwb_gt`、`search_alpha_gt` 及调试元信息。
  - 新增验证脚本 `script/verify_otb100_uwb_pipeline.py`，可从 `OTB100_UWB` 数据集真实读取，经 `UWBTrackingSampler + UWBProcessing` 全链路采样，打印字段类型/shape，并可视化 bbox、`uwb_seq`、`uwb_gt`，支持强制水平翻转验证。
- [x] 6. 已检查并确认 `lib/models/ugtrack/ugtrack.py` 已对齐：模型模块按 `UGTrack` 封装、`forward_uwb`、`forward_tracker`、`build_ugtrack` 组织，stage1 只构建 UWB 分支，stage2 构建并复用 OSTrack tracker 分支，整体结构节奏贴近 `build_ostrack`。
- [x] 7. 已检查并确认 `lib/train/actors/ugtrack.py` 已对齐：Actor 保持 `__call__ -> forward_pass -> compute_losses` 风格，stage1 负责 UWB 分支损失，stage2 复用检测损失与 heatmap 监督，并在 `status` 中保留 UWB 统计信息。

## 当前结论与遗留问题

### 数据链路结论

```text
OTB100_UWB -> UWBTrackingSampler -> UWBProcessing -> Actor 的数据组织已经打通
UWB 坐标已经改为跟随 bbox / crop / flip 的统一几何处理
可通过 verify_otb100_uwb_pipeline.py 做可视化和 shape 验证
```

### 训练启动侧遗留问题

```text
当前训练失败的主要问题不在数据处理，而在 checkpoint 加载逻辑
train_script_ugtrack.py 默认使用 load_latest=True
当当前实验目录下已有旧结构 checkpoint 时，会先尝试恢复它
旧 checkpoint 中仍是 uwb_encoder / uwb_alpha_head / uwb_pred_head 命名
而当前模型已改为 uwb_branch.encoder / uwb_branch.alpha_head / uwb_branch.pred_head
因此在训练开始前就会因 state_dict 键名不兼容而退出
```

### 下一步优先处理

```text
优先完成第 8 项：梳理 load_latest / load_previous_ckpt / OSTrack pretrained 的优先级
避免 stage1 新实验或 stage2 首次训练时误加载当前目录下的旧 checkpoint
```
