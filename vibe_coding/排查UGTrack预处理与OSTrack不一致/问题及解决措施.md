# 问题及解决措施

## 2026-05-01

| 问题 | 解决措施 |
| --- | --- |
| UGTrack 与 OSTrack 的 `masked_search` 第一块 search crop 不一致 | 验证发现 `sample_target()` 本身逐像素一致，差异来自 UGTrack 的 `self.state` 在后续帧已漂移 |
| UGTrack 测试只加载了约 1.7MB 的 UWB-only checkpoint，没有加载 370MB 的 OSTrack 视觉 tracker checkpoint | 修改 `build_ugtrack()`，让 `training=False` 时也加载配置中的 OSTrack checkpoint |
| debug yaml 未显式指定视觉 tracker checkpoint | 在 `uwb_tcn_residual_seq10_ep10_test.yaml` 和 `uwb_tcn_residual_seq10_ep10_prune_test.yaml` 中加入 OSTrack ViT-B base checkpoint 路径 |
| 需要遵守不污染官方 OSTrack | 本次只修改 UGTrack 构建逻辑和 UGTrack yaml，未修改官方 OSTrack backbone/tracker 流程 |
