# 问题及解决措施

## 2026-05-01

- 问题：原 CE 实现通过把 UWB token 放在 template 与 search 之间，并扩展 `global_index_t` 来避免 UWB token 被 CE 当作 search 剪枝，和 `UGTrack.md` 要求的 `[Template, Search, UWB]` 不一致。
- 措施：在 `VisionTransformerCEUWB` 中新增 search-only CE 候选消除逻辑，保持物理顺序为 `[Template, Search, UWB]`，但候选消除只作用于中间 search 段，末尾 UWB token 参与注意力且不进入剪枝候选。
- 补充问题：OSTrack head 固定取最后 256 个 token 作为 search 网格，若 CE 最终输出保留 UWB token，会导致最后 256 个 token 混入 UWB。
- 补充措施：CE 最终输出前丢弃 UWB token，只返回 `[Template, Search_restored]`。
- 验证：`python -m py_compile lib\models\ugtrack\vit_ce_uwb.py` 通过；张量级检查确认末尾 UWB token 被保留。
- 注意：张量级检查退出时 wandb 临时目录清理出现 Windows 权限错误，未影响检查结果和仓库文件。
