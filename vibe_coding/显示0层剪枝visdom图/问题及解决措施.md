# 问题及解决措施

## 问题

- 原 debug 可视化同时显示 `search_before_pruning`，不符合当前只看第 0 层剪枝效果的需求。
- 两个目标测试配置中 `UWB_PRUNE_ENABLE=False`，不会产生 `uwb_layer0_removed_indexes_s`，因此 Visdom 中无法显示第 0 层剪枝图。

## 解决措施

- 删除 `search_before_pruning` 的 Visdom 注册与本地图片保存逻辑。
- 开启目标测试配置中的第 0 层 UWB 剪枝，使 debug 输出 `uwb_pruned_search_layer0`。
