以后遇到要执行训练的时候，进行下面的操作：
- 写一个小的冒烟测试得出最佳的batch_size和num_workers
- 根据log进行估算当前batch_size和num_workers执行这个训练的单epoch时间
- 根据真正要实行的yaml估算完整训练所要花费的时间

## UGTrack 训练前检查

训练前先确认当前任务是“新实验”还是“断点续训”：

- 新实验：使用新的 config 名称，或确保 `output/checkpoints/train/<script>/<config>/` 下没有旧 checkpoint。
- 断点续训：复用同一个 config，训练脚本会自动加载当前实验目录下最新 checkpoint。
- Stage-2 初始化：必须传入 `--script_prv ugtrack --config_prv <stage1_config>`，用于加载阶段一 UWB 权重。

检查当前实验是否已有 checkpoint：

```powershell
Get-ChildItem .\output\checkpoints\train\ugtrack\<config_name>\*.pth.tar
```

Stage-2 的两种推荐初始化：

```powershell
# UWB 加载 Stage-1；OSTrack 加载 MAE ViT，head 随机初始化
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_uwb_conv1d_token --script_prv ugtrack --config_prv uwb_conv1d --save_dir ./output --mode single --use_wandb 0

# UWB 加载 Stage-1；OSTrack 加载已训练 OSTrack tracker，包含 ViT 和 head
python tracking/train.py --script ugtrack --config vitb_256_mae_ep300_ostrack_uwb_conv1d_token --script_prv ugtrack --config_prv uwb_conv1d --save_dir ./output --mode single --use_wandb 0
```
