### 3.2 数据集接口实现

#### 创建文件: `lib/test/evaluation/otb100uwbdataset.py`

**关键特性**:
1. ✅ 继承 `BaseDataset` 基类
2. ✅ 支持 train/val/test 三种 split
3. ✅ 自动扫描序列目录
4. ✅ 适配 8 位数字图像编号
5. ✅ 仅使用视觉部分（groundtruth.txt）

**核心方法**:

```python
class OTB100UWBDataset(BaseDataset):
    def __init__(self, split='test'):

    def get_sequence_list(self):
        """返回所有序列列表"""
        
    def _construct_sequence(self, sequence_info):
        """构建单个序列对象"""
        
    def _get_sequence_info_list(self):
        """自动检测序列结构"""
```

#### 注册数据集 (`lib/test/evaluation/datasets.py`)

```python
dataset_dict = dict(
    otb100_uwb=DatasetInfo(
```

### 3.3 关键技术点

#### 图像编号格式

- **OTB100_UWB**: 8位数字 (`00000001.jpg`)
- **标准 OTB**: 4位数字 (`0001.jpg`)

**解决方案**: 自动检测编号位数

```python
sample_img = img_files[0]
nz = len(sample_img.split('.')[0])  # 自动检测为 8
```

#### Split 动态加载

通过 `split` 参数支持不同子集：

```python
# 测试集
dataset = OTB100UWBDataset(split='test')

# 训练集
dataset = OTB100UWBDataset(split='train')
```

## 五、性能优化

### 5.1 评估速度优化

#### 问题发现

初始评估使用 `--threads 8`，FPS 仅为 10-13，耗时 40-60 分钟。

#### 优化方案

**修改前**:
```powershell
python tracking/test.py ... --threads 8 --num_gpus 1
# FPS: 10-13
# 时间: 40-60 分钟
```

**修改后**:
```powershell
python tracking/test.py ... --threads 1 --num_gpus 1
# FPS: **80**
# 时间: 约30秒
```

#### 优化效果

| 配置 | FPS | 总时间 | 提升 |
|------|-----|--------|------|
| `--threads 8` | 10-13 | 40-60分钟 | - |
| `--threads 1` | **80** | **约30秒** | **+523%** ⚡⚡⚡⚡ |

**原因分析**:
1. Windows 下多进程开销极大
2. 多线程导致严重的上下文切换和同步开销
3. 单线程避免了进程间通信成本
4. GPU 推理本身已足够快，无需多线程并行

#### 1. Windows Python 路径问题

#### 2. PyTorch 2.0 兼容性

**问题**: `ModuleNotFoundError: No module named 'torch._six'`

**原因**: PyTorch 2.0 移除了 `torch._six` 模块

**解决**: 修改导入语句
```python
# 旧代码
from torch._six import container_abcs

# 新代码
import collections.abc as container_abcs
```

#### 3. 依赖安装加速

**技巧**: 使用清华镜像源
```powershell
pip install package_name -i https://pypi.tuna.tsinghua.edu.cn/simple
```
